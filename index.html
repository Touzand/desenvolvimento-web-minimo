# A Terminal

Se tem uma coisa que você deve saber usar quando se fala em desenvolvimento de software, é o terminal.

Uma das coisas que sempre foi criticada pelo Windows no desenvolvimento de software é a falta de um bom terminal, embora hoje existam maneiras de melhorar isso.

A primeira coisa é instalar o terminal do Windows. este terminal é desenvolvido pela microsoft e permite abrir o CMD, o Powershell ou o shell do Azure, além de outros. É facilmente personalizável com transparências, fundo, cores e perfis.

A próxima coisa seria melhorar o powershell. você pode instalar um shell melhor como o OhMyPosh, que permite que você tenha um programa semelhante ao zsh no Windows e também é facilmente personalizável.

Mas se no final você não gostar do powershell, você tem a opção de instalar o Windows subsystem for linux. isso é basicamente linux dentro do windows. desta forma você pode ter sistemas como ubunto, kali linux ou debian e todos os seus programas como o próprio bash interagindo dentro do seu sistema linux. Ele ainda permite que você execute GUIs do Linux dentro do Windows.

# Vim - Primeiro contato

O Vim é um editor de texto, muito famoso porque é para terminal e porque possui combinações de teclas muito poderosas, embora complexas de aprender. Ele vem como uma evolução para outro editor muito semelhante, usado no UNIX, chamado Vi.

Vim, embora tenha um entorno gráfico, onde é mais utilizado é diretamente do terminal. Para muitos ( incluindo eu ) isso é uma vantagem porque eles podem trabalhar de uma conexão SSH para um servidor remoto. Outro ponto de diferenciação do Vim é que ele é centrado no teclado. Embora existam config e plugins para fazer funcionar com o mouse, a graça é usá-lo com o teclado porque assim você economiza tempo.

E foi aí que descobri o touchTyping, mas podemos falar sobre isso em outro blog

A primeira vez que instalei o Vim no meu terminal, fiquei completamente perdido (normal) já enfrentando o editor você tem que aprender coisas e comandos para poder se mover minimamente e rolar pelo editor como os modos do vim, teclas, linha de comandos do vim e claro... você tem que aprender a sair do vim (spoiler: você terá que assistir um tutorial) mas realmente instalar vim é muito facil

Se você estiver no Ubuntu/Debian, basta executar em um terminal:

sudo apt install vim

Se você estiver no Fedora:

sudo dnf install vim

Se você estiver no Arch Linux:

pacman -S vim

Se você preferir a versão snap:

sudo snap isntall vim-editor --beta

No Windows, se você estiver procurando pelo .exe do vim, poderá encontrá-lo na página de lançamentos. Clique em Ativos e você encontrará o arquivo .zip para Win64.

# Vim vs Nvim - existe uma diferença?

Para começar, o Vim é desenvolvido principalmente por seu criador, embora seja um projeto de código aberto e todos possam contribuir, quem decide e tem o controle de todo o projeto é seu criador. O Neovim é um projeto mais recente que o Vim, é mais voltado para a comunidade e seria apenas isso.

# Vim - Configuração global

Uma vez instalado o vim ou o nvim, você deve configurá-lo para funcionar como um editor de código para programação, pois por enquanto é apenas um editor de texto (como um bloco de notas, mas com esteróides) e por enquanto está assim...

Quando você começa a configurá-lo, você tem duas opções, configurá-lo usando VimScript ou Lua. dentro da comunidade há uma pequena disputa sobre esta questão. sobre se o vimScript é melhor ou lua é melhor. mas a verdade é a seguinte; Lua é mais ergonômica para escrever programas/plugins, mas o vimscript é mais ergonômico para uso interativo (e talvez arquivos de configuração menores).

E aqui estamos falando de conforto ao configurar o vim ou programar um plugin. vimScript é muito repetitivo e chato, e lua é mais confortável e... divertido de usar... poderíamos dizer. mas a verdade é que, se você não é um vimmer, não deve se preocupar muito com isso, ainda não, o que você precisa se preocupar é que seu vim funcione corretamente, então vamos dar uma olhada em algumas configurações básicas . Você escreverá essas configurações em um arquivo .vimrc na ruta 

~/.vimrc                     " Pra vim

Ou na ruta

~/.config/nvim/init.vim      " Pro Nvim

se falarmos sobre o vim, caso você esteja no windows estaria no

`C:/Users/<SEU_USUARIO>/_vimrc`

No Mac deve estar aqui, embora eu não tenha certeza porque não uso Mac: 

/usr/share/vim/vimrc

Agora vou te falar sobre algumas configurações que recomendo para começar no vim. então você pode investigar por conta própria se deseja adicionar mais ou remover alguns. Acredite, é um mundo enorme que você pode explorar para enriquecer sua configuração do vim.

VimScript 
syntax on            " Ativar destaque de sintaxe "
set nu               " Ativar linha de numeros "
set tabstop=4        " Mostrar guia existente com 4 espaços de largura "
set softtabstop=4    " Mostrar guia existente com 4 espaços de largura "
set shiftwidth=4     " Ao recuar com '>', use 4 espaços de largura "
set expandtab        " Ao pressionar tab, insira 4 espaços "
set smarttab         " inserir tabulações no início de uma linha de acordo com a largura do deslocamento "
set smartindent      " inserir tabulações no início de uma linha de acordo com a largura do deslocamento "
set hidden           " Esconde o buffer atual quando um novo arquivo é aberto "
set incsearch        " pesquisa incremental "
set ignorecase       " Ingore case in search "
set smartcase        " Considere maiúsculas se houver um caractere maiúsculo "
set scrolloff=8      " Número mínimo de linhas a manter acima e abaixo do cursor "
set colorcolumn=100  " Desenha uma linha na linha fornecida para manter-se ciente do tamanho da linha "
set signcolumn=yes   " Adicione uma coluna à esquerda. Útil para forro "
set cmdheight=2      " Dê mais espaço para exibir mensagens "
set updatetime=100   " Tempo em milissegundos para considerar as mudanças "
set encoding=utf-8   " A codificação deve ser utf-8 para ativar os ícones de fontek "
set nobackup         " Não faça backup de arquivos "
set nowritebackup    " Não faça backup de arquivos "
set splitright       " Crie as divisões verticais à direita "
set splitbelow       " Crie as divisões verticais à direita "
set autoread         " Atualize o vim após a atualização do arquivo de fora "
set mouse=a          " Ativar suporte a mouse "
filetype on          " Detecte e defina a opção de tipo de arquivo e acione o evento FileType "
filetype plugin on   " Carregue o arquivo de plug-in para o tipo de arquivo, se houver "
filetype indent on   " Carregue o arquivo de plug-in para o tipo de arquivo, se houver "

# Vim - Plugins

Ao falar de Plugins, devemos procurar ferramentas para instalá-los. No meu caso eu uso o Vim-Plug do junegunn, mas também existem outras ferramentas semelhantes, como o Packer.vim.

O Vim-Plug é um gerenciador de plugins gratuito, de código aberto e minimalista que pode instalar ou atualizar plugins em paralelo. cria clones de repositório de cada plugin separadamente para minimizar o uso do espaço em disco e o tempo de download

Eu recomendo que você vá no repositório do github do vim-plug para aprender como se instala e entender melhor como usar e trabalhar com ele. tendo o vimplug, podemos ir ao nosso arquivo de configuração global do vim e adicionar essa estrutura a ele. Deve ser algo parecido com isso: 

call plug#begin()

// Seus plugins aqui

call plug#end()

E com esta estrutura podemos começar a instalar plugins. escrevendo algo como esses três exemplos 

call plug#begin()

//A árvore NERD será carregada na primeira invocação do comando NERDTreeToggle"
Plug 'preservim/nerdtree', { 'on': 'NERDTreeToggle' }

//Parênteses, colchetes, aspas, tags XML e muito mais
Plug 'tpope/vim-surround'

//Escrita sem distrações no Vim.
Plug 'junegunn/goyo.vim'

call plug#end()

E já tendo eles em nossa lista de plugins você notará que ainda não há alteração. isso ocorre porque para o vimplug funcionar, precisamos dizer a ele para fazê-lo com alguns comandos como: 

:PlugInstall    "Instalar plug-ins"
:PlugUpdate     "Instalar ou atualizar plugins"
:PlugClean      "Remova os plugins não listados (a versão bang será limpa sem aviso)"
:PlugUpgrade    "Atualize o próprio vim-plug"
:PlugStatus     "Verifique o status dos plug-ins"
:PlugDiff       "Examine as alterações da atualização anterior e as alterações pendentes"

Então agora, como indiquei acima, precisamos digitar :PlugInstall na linha de comando do vim para instalar nossa listas de plugins. deixe carregar, feche o vim e abra novamente ou só escreva :so % e pronto temos nossos plugins

# Min browser

Já falamos sobre um editor de texto para poder trabalhar seguindo essa ideologia. mas agora vamos falar sobre um navegador mínimo muito interessante que eu pessoalmente uso diariamente.

Você provavelmente conhece alguns navegadores mais famosos como Google chrome, Opera, Firefox, Microsoft Edge, Safari entre outros, é compreensível.

Vamos falar um pouco sobre o Google Chrome que é líder no ranking de navegadores. hoje, é utilizado por mais de 3.000 milhões de usuários, especificamente, por 3.378.967.819 pessoas, o que equivale a 64,91% dos internautas. isso é incrível, possivelmente você está lendo este blog do chrome. Isso é um tanto surpreendente, especialmente considerando a magnitude do ecossistema de aplicativos e serviços online do Google, incluindo YouTube, Drive, Agenda, Documentos, Maps e Earth, entre outros.

Mas há um problema. e eu sei que o chrome tem muitos outros problemas, mas vou focar apenas em um deles, o que mais me importava naquele momento, seu alto consumo de memória RAM. se você pode usar o chrome sem problemas, vá em frente, mas hoje quero te apresentar uma alternativa mínima

Apresento a você o Min Browser, um navegador focado na simplicidade. Min é um navegador rápido e minimalista que protege sua privacidade. Inclui uma interface projetada para minimizar distrações e funçôes como:

  • Pesquisa de texto completo das páginas visitadas
  • Bloqueio de anúncios e rastreadores
  • Visualização do Leitor Automático
  • Tarefas (grupos de guias)
  • Integração do gerenciador de senhas

Entre várias outras funções. Eu recomendo que você dê uma olhada

Claro que o navegador Min também tem suas desvantagens. como vulnerabilidade a ataques virtuais (hacking). Vulnerabilidade que o Chrome minimiza a cada atualização
